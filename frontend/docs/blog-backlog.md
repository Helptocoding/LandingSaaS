# Backlog editorial del Blog SaaslyVet

Banco de ideas priorizado para los próximos 6+ meses. Cada idea es un `.md` que cumple el frontmatter estándar (`src/content.config.ts`); publicar no requiere tocar arquitectura.

**Cómo leer la tabla**

- **Intención:** `Informacional` (aprender) · `Comercial` (comparar/evaluar) · `Transaccional` (listo para actuar).
- **Dificultad:** esfuerzo estimado para posicionar (competencia + profundidad). `Baja` / `Media` / `Alta`.
- **Prioridad:** `P1` = escribir primero (alto valor / intención cercana a la compra o gran volumen). `P2` = segundo trimestre. `P3` = relleno de autoridad temática.
- **CTA:** `trial` (registro/prueba), `demo` (modal de demo), `sales` (cadenas/multi-sucursal).

Orden de ejecución sugerido: **P1 comerciales/transaccionales → P1 informacionales de alto volumen → P2 → P3**. Cada pilar debe tener 4–6 artículos para consolidar el clúster antes de abrir el siguiente.

## Notas de estrategia de clúster

1. **Interlinking obligatorio.** Cada artículo enlaza (a) a su página pilar de categoría y (b) a 2–3 artículos hermanos del mismo clúster. Usar el campo `relatedSlugs` cuando se quiera forzar el enlazado; si se deja vacío, el sistema autocompleta por categoría.
2. **Consolidar antes de expandir.** Publicar 4–6 artículos por clúster antes de abrir el siguiente, para que Google reconozca la autoridad temática.
3. **Un `featured` a la vez.** Solo un artículo con `featured: true` (sale como destacado en `/blog`). Rotarlo hacia el contenido de mayor conversión del momento.
4. **FAQ siempre.** Incluir 3–5 preguntas en `faq` por artículo: alimenta el schema `FAQPage` y captura *featured snippets* y *People Also Ask*.
5. **Cadencia sugerida:** 2–4 artículos/mes. En 6 meses ≈ 18–24 artículos → los 3 clústeres P1 (Software, Automatización, Administración) completos + inicio de los demás.
